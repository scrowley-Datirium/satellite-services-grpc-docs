scidap-satellite documentation

GRPC-Service

This is the Main Satellite Service that manages the starts all of the other services. It is essentially a wrapper for pm2 that manages the instances with Finite State Machines

How to Test`

When testing we pass in two more Environment Variables

  • SAT_ID which is the id of the satellite so we can test we've found the correct one
  • TEST_FILE which is where we read the configs from, this might become extraneous eventually npm run test will run the standard testing packing npm run test:logs will run the tests but show logging output npm run test:latest will only run test that has a tag latest npm run test:travis will run travis tests(they're slightly different) also this is currently deprecated

Environment Variables

# Whether to run in a testing environment or not
TEST=true
# What level of logs to show (defaults to info)
LOG_LEVEL=debug
# Whether to show logs, only relevant in testing
LOGS=true
# What is the environment (which settings file to load)
API_ENV=local
# Optional, if this is set we don't kill all the services run by pm2 with the namespace
SKIP_KILL=true
